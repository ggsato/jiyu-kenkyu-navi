# 採用済み判断

この文書は、Issue ベースで採用済みになった重要判断を短くまとめた一覧である。  
長い議論の本文は各 Issue に残しつつ、採用事項だけをここで追えるようにする。

## Issue #1

- 問い・記録・振り返りのループを回し続けられることを重視する
- 家族内の軽量ユーザー切り替えを持つ MVP とする

## Issue #2

- AI は問いの作者ではなく、本人の問いの芽を整える役に寄せる
- `next_curiosity_text` のような任意入力を持つ
- 問い選択後に「記録項目を整える」ステップを持つ
- 記録項目は役割つき候補 + 本人選択にする
- 本人が手動で記録項目を追加できる
- 記録項目は観測値を優先し、集計値を直接持ち込まない
- 記録項目には「何を見るためか」と「あとでどう使うか」を持たせる

## Issue #3

- Issue は意思決定ログとして残す
- 現行仕様の正本は文書に再統合する
- 壊してはいけないことは短い別文書で明文化する
- AI に渡す最小コンテキストと文書更新ルールを定義する

## Issue #5

- 記録項目設計の重心を、問い依存より願い依存へ寄せる
- 同じ願いのもとでは、比較的安定した観測骨格を持つ
- 小さな問いは、その観測骨格のどこを見るかを決める焦点設定として扱う
- 問い選択後の「記録項目を整える」ステップは維持する
- ただし毎回ゼロから専用項目を作るのではなく、既存項目の再利用・追加・細分化を行う
- AI は毎回新規項目を設計する役より、長く役立つ観測軸と今回注目する項目を提案する役に寄せる
- 問いごとの `使う / 使わない` は履歴として保持し、未選択項目も願いの観測骨格には残す
- MVP では、項目関係のうち継承と細分化を扱う

## Issue #9

- 記録の時間情報は任意ではなく基本項目とする
- MVP v1.2 の入力方式は `日時` を基本とする
- 記録一覧と振り返りは時間順に扱えることを前提にする
- 記録一覧の表示順は保存時刻ではなく `recordedAt` を基準にする

## Issue #10

- MVP v1.2 ではキャラクター画像は固定のデフォルト表現を維持する
- 写真ベース表現は将来検討とし、安全性・家族運用・保存方針の整理を先に行う

## Issue #11

- 細分化は、選択肢追加ではなく観測の解像度を上げる提案だけを採用する
- 細分化候補には `why` と `howToUse` を持たせ、何が見えるようになるかを説明する

## Issue #12

- active な問いでは、問いを作り直さずに記録項目を後から編集できる
- 保存済み記録は書き換えず、同じ key を通じて見方だけを更新する
- 一度も採用されていない未使用候補は観測骨格に残さない

## Issue #13

- 継続提案では `unit` `type` `options` `howToUse` を既存項目メタデータとして参照する
- 再利用優先度は直前の問いだけでなく、長期の利用回数と継続履歴も考慮する
