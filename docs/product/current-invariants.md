# Current Invariants

この文書は、今後の改良でも壊してはいけない条件を短くまとめたものである。  
Codex / AI に作業を依頼するときは、まずこの文書を読む。

## 1. 問い生成

- AI は問いの作者ではなく、本人の問いの芽を整える役に寄せる
- 本人入力がある場合、その言葉から離れすぎない
- 問いは一回で試せて、観察・記録可能であることを優先する

## 2. 記録項目

- 問い選択後に、記録項目を整えるステップを持つ
- ただし整える対象は、毎回ゼロから作る問い専用項目ではなく、願いにひもづく観測骨格の焦点設定・詳細化・追加に寄せる
- 記録項目は観測値を優先し、集計値や評価値を直接持ち込まない
- AI 提案は初期案であり、本人が選び直せる
- 本人が手動で項目を追加できる
- 問いごとの項目選択では、使わない項目も観測骨格から消さず、問いごとの選択状態として保持する
- 記録項目には意味説明が必要である
- 各項目は「何を見るためか」と「あとでどう使うか」を持つ

## 3. 継続支援

- 記録しかできない時期も前進として支える
- 新しい問いが出ないことを失敗とみなさない
- 次の一歩は、新しい問いでなくてもよい

## 4. 文書運用

- Issue は正本ではない
- 実装に影響する重要判断は、正本文書へ昇格させる
