# 自由研究ナビ

この文書は、自由研究ナビがどのような役割を担うプロダクトかをまとめた文書である。

## 1. このアプリは何をするか

自由研究ナビは、自由研究を定義するためのアプリではない。  
自由研究の営みを支えるための道具である。

このアプリの中心機能は次の三つである。

- 願いを出発点として受け取ること
- 願いに対して小さな問いの焦点を整えること
- 願いにひもづく観測骨格の上で、問いを回しながら現在地と次の一歩を見えるようにすること

自由研究ナビは、答えを出すアプリではなく、問いを育てるアプリである。

## 2. 設計の前提

### 2.1 出発点はテーマではなく願い

アプリは「研究テーマを入力してください」から始めない。  
「何を願っているのか」を受け取るところから始める。

### 2.2 一度で正しい問いを作る必要はない

最初の問いは粗くてよい。  
仮の問いを立て、小さく試し、記録し、振り返りながら問いを育てる。

### 2.3 現在地が見えれば前に進める

利用者が止まりやすい大きな理由は、「何も進んでいない気がする」ことである。  
自由研究ナビは、距離・針路・履歴を見えるようにして、前進感を支える。

## 3. アプリの基本ループ

1. 願いを言葉にする
2. 小さな問いを整える
3. 今回の目的を選ぶ
4. 試す・観る・集める
5. 記録する
6. 振り返る
7. 次の一歩を決める

このループが何度も回ることで、問いが育ち、理解が深まり、行動が変わっていく。

## 4. AI の役割

AI は問いの作者ではない。  
AI の役割は、本人の願いや問いの芽を読み取り、それを今回回せる大きさへ整えることである。

また、問いを選んだあとには、その問いを見るためにどんな記録項目が必要かを提案し、利用者が理解・選択できるよう支える。

ただし、ここでいう記録項目は毎回ゼロから作る問い専用の器ではない。  
同じ願いのもとで比較的安定して使える観測骨格を前提にしながら、今回の問いではどこを見るか、どこを細かく分けるかを提案する役に寄せる。

## 5. 記録設計の考え方

記録項目は、ただ入力させるための項目ではない。  
願いに向かう中で世界をどう観るかを支える、比較的安定した観測骨格である。

小さな問いは、その観測骨格のどこに注目するかを決める。  
問いが育つときは、記録項目を毎回全交換するより、既存項目を引き継ぎ、必要なところだけ細かく分けたり足したりする方が自然である。

したがって、記録項目は次の条件を満たすべきである。

- 観測値である
- 一回ごとに直接記録できる
- 後で比べたり見返したりできる
- 何を見るための項目かが分かる

記録項目の意味が分からない状態は、問いの意味が分からない状態に近い。  
そのため、項目名だけでなく、何を見るための項目か、あとでどう使うかも明示する。

## 6. 記録しかできない時期を支える

自由研究ナビの価値は、問いを生むことだけではない。  
問いが動かない時期や、記録しかできない時期でも、前進感を失わずに続けられることが重要である。

そのために、

- 記録が前進の材料であることを示す
- 今は記録の時期だと分かるようにする
- 次にやる最小の一歩を返す

ことを重視する。
